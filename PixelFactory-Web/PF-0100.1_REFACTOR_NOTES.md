# PF-0100.1 — Palette Lab ProcessingService Extraction

## What changed

`PixelFactory-Web/backend/main.py` had ~600 lines of Palette Lab pixel
processing (pixel-size detection, quantize, smart downscale, pixel snap,
palette normalize, morphology/orphan/jaggy/edge cleanup, and their stat
helpers) living as bare module-level functions inside the FastAPI route file.

Those 18 functions now live in:

`PixelFactory-Web/backend/services/palette_service.py`

as static methods on a new `PaletteLabService` class, matching the same
pattern already used by `AssetService`, `WorkspaceService`, and
`ExportService` in that folder.

`main.py` now creates one instance at startup:

```python
palette_lab_service = PaletteLabService()
```

and every call site that used to say `_quantize_rgba(...)` now says
`palette_lab_service.quantize_rgba(...)` (same for all 18 functions).

`_read_image` and `_png_response` stayed in `main.py` — those are
request/response glue, not Palette Lab domain logic, so they didn't move.

`main.py`: 1,359 lines → 804 lines.

## What did NOT change

- No algorithm changed.
- No default value, tolerance, threshold, or strength curve changed.
- No API route signature, request field, or response shape changed.
- No frontend file touched at all — `app.js`, `styles.css`, `index.html`
  are byte-identical to what you had.

This was checked, not assumed: every moved function was run against the
pre-refactor code on four real images pulled from your own
`PixelFactory-Projects/default/Incoming` and `Accepted` folders (two tiles,
two characters), across the parameter combinations your `/api/process`
route actually uses (pixel snap with/without quantize, smart downscale,
palette normalize, morphology/orphan/jaggy/edge cleanup at 0.5 strength,
override and auto pixel-size detection). All 60 checks came back
byte-for-byte identical between old and new code.

## How to test it yourself

1. Drop this folder in place of your current project folder (same
   structure, so it's a straight replace like you've been doing).
2. `cd PixelFactory-Web && pip install -r requirements.txt` (only needed if
   you're on a fresh env).
3. Run the server the way you normally do (`uvicorn backend.main:app` or
   your usual launch command) and open the UI.
4. Load a candidate image into Palette Lab and run your normal repair passes
   — Pixel Snap, Smart Downscale, Palette Normalize, the cleanup toggles,
   export. Everything should look and behave exactly like before, because
   the underlying code is unchanged, just relocated.
5. If anything errors on startup, it's almost certainly an import path
   issue — check that `backend/services/palette_service.py` exists and that
   `main.py`'s import line near the top reads:
   `from backend.services.palette_service import PaletteLabService`.

## If something breaks

Tell me exactly what you did (which button/toggle, which image) and paste
the traceback/error text. Since this pass didn't touch any algorithm, a
break here almost certainly means an import/wiring mistake on my end, not a
processing regression — that narrows down where to look fast.
