# PF-0010.1 - Asset State Clarity Patch

Status: Ready for testing

## Changes

- Incoming and accepted assets now show clearer badges.
- Inspector shows current image path, original incoming path, accepted copy path, and project root.
- Accepted assets serve the accepted copy as the active image when available.
- New generated assets no longer lose their generated default name because of an empty caller-provided name.
- Character generation is locked to safe square sizes: 512, 768, 1024.
- Backend rejects non-square / unsafe sizes to prevent broken latent-size workflow output.

## Notes

Incoming assets are generated candidates. Accepted assets are copied into the Accepted folder and preserve metadata/history.
