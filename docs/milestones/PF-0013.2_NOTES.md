# PF-0013.2 Export Route / Packaging Stability

This pass fixes the export test failure where the UI showed `Not Found` when exporting from the Asset Browser or Exporter page.

Root cause: the previous handoff ZIP contained a nested stale `PixelFactory/` copy alongside the live project files. Running the app from the stale nested copy could show older backend routes while newer frontend files existed elsewhere, producing 404 `Not Found` on export endpoints.

Actions:
- Removed the nested stale project copy from the handoff package.
- Kept one canonical repo root.
- Verified backend export service can export accepted assets.
- Updated visible app milestone to PF-0013.2.

Expected export endpoints:
- `GET /api/exports`
- `POST /api/assets/{asset_id}/export`
- `POST /api/exports/{target}/accepted`

Important restart note: after replacing files, stop and restart the FastAPI server. Browser refresh alone cannot load new backend routes.
